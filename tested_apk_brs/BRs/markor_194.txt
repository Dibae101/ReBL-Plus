App Name: Markor

Bug Report Title:
App crashes when rotating device during filesystem dialog

Bug Report Issue:
Markor crashes with NullPointerException when device is rotated while the filesystem browser dialog is open.

### DESCRIPTION
The FilesystemDialog fragment doesn't properly save and restore its state during configuration changes, resulting in a null Options object.

### STEPS TO REPRODUCE
1. Open Markor app
2. Create or open a document
3. Click "Insert image" option
4. Click "BROWSE FILESYSTEM" button
5. Enable auto-rotate on device
6. Rotate the device (portrait to landscape or vice versa)
7. App crashes with NullPointerException

### EXPECTED BEHAVIOR
The filesystem dialog should handle device rotation gracefully and redraw without crashing.

### ACTUAL BEHAVIOR
App crashes with NullPointerException when trying to access FilesystemDialogData$Options.cancelButtonEnable field on a null object.

### ENVIRONMENT
Markor version: v0.3.2
Android version: 6.0.1
Device: Nexus 5
