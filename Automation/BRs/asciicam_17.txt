App Name: AsciiCam

Bug Report Title:
java.lang.IndexOutOfBoundsException: Invalid index 0, size is 0

Bug Report Issue:
AsciiCam crashes with IndexOutOfBoundsException when deleting a picture after rotating the device in the library view.

### STEPS TO REPRODUCE
1. Open AsciiCam app
2. Take a photo and return to library
3. Click the duplicate/gallery icon and select the top-leftmost picture
4. **Important**: Make sure there is only ONE picture available in the library
5. Rotate your phone (auto-rotate must be enabled)
6. Click "DELETE PICTURE" button
7. App crashes with IndexOutOfBoundsException

### EXPECTED BEHAVIOR
The picture should be deleted without crashing, or the app should handle the rotation gracefully.

### ACTUAL BEHAVIOR
App crashes with:
```
java.lang.RuntimeException: Unable to resume activity
Caused by: java.lang.IndexOutOfBoundsException: Invalid index 0, size is 0
at java.util.ArrayList.get(ArrayList.java:308)
at com.dozingcatsoftware.asciicam.LibraryActivity.onActivityResult(LibraryActivity.java:124)
```

### ENVIRONMENT
Package: com.dozingcatsoftware.asciicam
Version: Any version BEFORE v1.2.4 (bug fixed on March 25, 2018)
Recommended test version: v1.2.3 or earlier

### COMMENTS
Bug was reported on March 7, 2018 (Issue #17) and fixed on March 25, 2018 in PR#19.
Fixed in version 1.2.4 and later.
